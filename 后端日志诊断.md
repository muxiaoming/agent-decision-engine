# 后端日志诊断 - 正常流程说明

## 正常情况：每次请求调用 5-7 次大模型

投资决策流程会**按顺序**执行 6 个步骤，每个步骤调用一次大模型：

### 完整流程

```
用户发送: "我想投资科技股"
    ↓
Step 1: 问题感知 (Skills) → 调用大模型 1次
    ↓
Step 2: 知识检索 (RAG) → 调用大模型 1次  [可选]
    ↓
Step 3: 数据获取 (Tools) → 调用工具 3-5次 (股票、指数、风险等)
    ↓
Step 4: 推理分析 (Skills) → 调用大模型 1次
    ↓
Step 5: 决策生成 (Skills) → 调用大模型 1次
    ↓
Step 6: 流程编排 (Graph) → 调用大模型 1次  [可选]
    ↓
完成 → 发送 decision_complete 事件
```

### 大模型调用次数统计

| 步骤 | 名称 | 技能 | 大模型调用 | 工具调用 |
|------|------|------|-----------|---------|
| 1 | 问题感知 | Skills | 1次 | 0次 |
| 2 | 知识检索 | RAG | 1次 | 0次 |
| 3 | 数据获取 | Tools | 0次 | 3-5次 |
| 4 | 推理分析 | Skills | 1次 | 0次 |
| 5 | 决策生成 | Skills | 1次 | 0次 |
| 6 | 流程编排 | Graph | 1次 | 0次 |
| **总计** | | | **5-6次** | **3-5次** |

**每次请求总耗时**: 约 15-30秒

## 如果日志一直在重复

### 可能的原因 1: 前端不断重试

**症状**: 看到相同的流程重复执行多次

**检查**:
```bash
# 查看前端日志
# 应该只看到一次 "Decision params: {...}"
```

**解决**: 检查前端是否在不断调用 onSubmit

### 可能的原因 2: 后端某个步骤失败导致重试

**症状**: 某个步骤反复失败并重试

**检查**:
```bash
# 查看日志中是否有错误
grep "ERROR\|WARN" logs/app.log
```

**解决**: 检查 API 密钥是否有效

### 可能的原因 3: SSE 事件流没有正确结束

**症状**: 后端一直发送事件，没有 decision_complete

**检查**:
```bash
# 查看最后一条事件
curl -X POST http://localhost:8182/api/investment/decide/stream \
  -H "Content-Type: application/json" \
  -d '{"message": "test", "modelName": "deepSeekChatModel", "enableRAG": false, "enableTools": false, "enableGraph": false}' \
  | tail -5
```

**预期**: 最后应该有 `{"type":"decision_complete",...}`
